Conclusiones del Diagnóstico Táctico (Microservicio API)

A través de un análisis estático de código complementado con despliegue de pruebas reales en C# mediante xUnit y WebApplicationFactory, se logró auditar el controlador principal. Hemos refutado ciertas asunciones (detectando un hueco de autenticación) y validado las defensas nativas.

Área Evaluada Hallazgos Principales (xUnit + QA) Impacto y Acción Recomendada
Control de Acceso Grave: Las pruebas de Integración demostraron que /api/v1/clientes devuelve 200 OK sin requerir JWT (Falta atributo [Authorize] global). Hay fuga de datos. Alta. Inyectar [Authorize] de inmediato.
Protección SQLi y DB Éxito: Entity Framework Core bloqueó la inyección (TC-S13). La Base de Datos está segura de ataques clásicos al usar parametrización nativa. Ninguna. Defensas activas impecables.
Integridad XSS Advertencia: No contamos con middlewares AntiXss. Un usuario malicioso logró insertar etiquetas <script> directamente en el campo Nombre. Media. Instalar paquete HTMLSanitizer.
Validación Funcional Éxito: La paginación, filtros por nombre, y eliminación lógica trabajan al 100%. DataAnnotations responde con estrictos 400 Bad Request. Progreso exitoso. DTOs validados.
